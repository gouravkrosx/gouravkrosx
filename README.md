# Gourav Kumar

Software Developer at [Keploy](https://github.com/keploy/keploy), working on API testing infrastructure, deterministic replay, service virtualization, and developer tooling.

I spend most of my engineering time close to the runtime: network proxies, database protocols, mocks, replay correctness, eBPF-based capture, and JVM instrumentation. I like turning hard-to-reproduce production behavior into reliable tests that developers can run locally and in CI.

## What I Work On

- Building and maintaining [Keploy](https://github.com/keploy/keploy), an open-source platform for API, integration, and end-to-end testing.
- Improving replay correctness across HTTP, gRPC, MySQL, PostgreSQL, MongoDB, Kafka, and Docker-based workflows.
- Debugging protocol-level behavior in record/replay systems, including prepared statements, binary rows, decompression, mock matching, and test isolation.
- Building developer-facing CLI flows, documentation, CI checks, telemetry, and release-quality fixes.
- Exploring agentic developer workflows around testing, docs, sandboxes, and MCP-based automation.

## Keploy Contributions

Public GitHub search currently shows **280+ pull requests** and **340+ commits** across Keploy repositories. A few representative areas:

- **Replay correctness:** fixed MySQL stateful prepared-statement mock selection and BLOB handling in [keploy/keploy#4235](https://github.com/keploy/keploy/pull/4235).
- **V1 sandbox stability:** shipped multiple sandbox replay fixes across MySQL, mock DB, runner, and recorder drain flows in [keploy/keploy#4200](https://github.com/keploy/keploy/pull/4200).
- **CLI and product flows:** added the Community Edition upgrade banner for the OSS binary in [keploy/keploy#4228](https://github.com/keploy/keploy/pull/4228).
- **HTTP and gRPC capture:** added response comparison controls and capture-size safeguards in [keploy/keploy#3732](https://github.com/keploy/keploy/pull/3732).
- **Docs and install experience:** improved macOS installation guidance and Homebrew/curl setup paths in [keploy/docs#868](https://github.com/keploy/docs/pull/868).
- **SDK and ecosystem:** contributed to the Keploy Go SDK, docs, CI, examples, and developer onboarding.
